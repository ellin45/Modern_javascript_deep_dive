# 실행 컨텍스트

```
실행 컨텍스트는 소스코드를 실행하는 데 필요한 환경을 제공하고 코드의 실행 결과를 실제로 관리하는 영역이다.
실행 컨텍스트는 식별자(변수, 함수, 클래스 등의 이름)를 등록하고 관리하는 스코프와 코드실행 순서 관리를 구현한 내부 매커니즘으로, 모든 코드는 실행 컨텍스트를 통해 실행되고 관리된다.
식별자와 스코프는 실행 컨텍스트의 렉시컬 환경으로 관리하고 코드 실행순서는 실행 컨텍스트 스택으로 관리한다.
```

```
호이스팅

생성단계 : Execution Context 생성
선언문만 실행해서 Environment Record에 기록
실행단계 : 선언문 외 나머지 코드 순차적 실행
Environment Record 참조하거나 업데이트

선언

일시적 사각지대

```
